OAuth权限接入手册

1.应用接入申请

  • 在移动管理平台申请应用接入，获得client_id和client_secret，这两个值是后续进行应用接口调用的关键内容 ，接入应用需要保存好，作为后续登录的唯一认证id。如下图，依次点击工作台->工作台管理->新建进行应用创建:

  • 新建应用时需要填入相应的应用信息，包括应用名称、图标、平台类型、技术选型、应用首页、回调地址等信息，其中应用首页和回调地址由第三方应用提供。

    应用首页：web应用的首页地址。

    回调地址：第三方应用提供的回调页面地址，授权码会通过该回调地址返回给三方应用。

  • 应用新建成功后会将应用信息提示给创建者，该信息需要第三方应用保存好，后面获取登录用户信息时会用到。

2.应用权限获取接口

1. 查询指定项目属性

接口功能

获取制定项目的分类信息

URL

http://ip:port/oauth/authorize

支持格式

JSON

HTTP请求方式

GET

请求参数
参数 必选 类型 说明
name ture string 请求的项目名
type true int 请求项目的类型。1：类型一；2：类型二 。
返回字段
返回字段 字段类型 说明
status int 返回结果状态。0：正常；1：错误。
company string 所属公司名
category string 所属类型
接口示例

地址：http://www.api.com/index.php?name="可口可乐"&type=1

{
    "statue": 0,
    "company": "可口可乐",
    "category": "饮料",
}
